Planificador de eventos por Fecha/Hora en MySQL 5.1 o superior



MySQL provee una característica que nos permite ejecutar sentencias SQL o un Procedimiento almacenado a una hora determinada o cada cierto tiempo.

Por ejemplo, supongamos que un cajero automático dispone de un retiro de $ 1.000 diario por cuenta. El sistema, a las 00:00 de cada día debería actualizar el monto a retirar y establecer los $ 1.000 para el día que comienza. Para este ejemplo, es útil implementar Eventos ya que podríamos crear un evento recurrente que todos los días a las 00:00 se actualice el monto a retirar de cada cuenta.

Antes de comenzar, debemos setear nuestro MySQL para que acepte planificación de eventos, ejecutando la sentencia SQL siguiente:

SET GLOBAL event_scheduler = ON;

La sintáxis es la siguiente:

CREATE
[DEFINER = { user | CURRENT_USER }]
EVENT
[IF NOT EXISTS]
event_name
ON SCHEDULE schedule
[ON COMPLETION [NOT] PRESERVE]
[ENABLE | DISABLE | DISABLE ON SLAVE]
[COMMENT 'comment']
DO sql_statement;

schedule:
AT timestamp [+ INTERVAL interval] ...
| EVERY interval
[STARTS timestamp [+ INTERVAL interval] ...]
[ENDS timestamp [+ INTERVAL interval] ...]

interval:
quantity {YEAR | QUARTER | MONTH | DAY | HOUR | MINUTE |
WEEK | SECOND | YEAR_MONTH | DAY_HOUR | DAY_MINUTE |
DAY_SECOND | HOUR_MINUTE | HOUR_SECOND | MINUTE_SECOND}

Necesitamos un nombre para el evento “event_name”, una condición para que se cumple “ON SCHEDULE”, y una sentencia a ejecutar “DO”. Por ejemplo

Ejemplo 1:

CREATE EVENT e_ActualizaMonto
ON SCHEDULE EVERY 1 DAY STARTS ‘2009-01-01 00:00:00’
DO UPDATE cuentas SET monto_a_retirar = 1000

Supongamos que ahora nos piden además, que un cliente no puede retirar dinero más de 1 vez por hora. Una solución sería mantener un campo en la tabla ‘cuentas’ que nos diga si el cliente puede retirar dinero, entonces luego de una extracción, ese campo se cambia a “0” y creamos un evento que se ejecute 1 hora después de la extracción haciendo ese campo en “1”.
En algún lugar de un Procedimiento almacenado que procese la extracción, debemos crear el siguiente Evento:

Ejemplo 2:

UPDATE cuentas SET puede_retirar = 0 WHERE numero_cuenta = @num_cuenta

CREATE EVENT e_BloqueaExtraccion
ON SCHEDULE AT now() + INTERVAL 1 HOUR
DO UPDATE cuentas SET puede_retirar = 1 WHERE numero_cuenta = @num_cuenta

El evento del ejemplo 1 es recurrente, es decir, hasta que no lo eliminemos se ejecutara siempre. El del ejemplo 2, se ejecuta una sola vez y se elimina automáticamente.

Son muy simples y nos dan soluciones a muchos problemas, y muy potentes si los utilizamos conjuntamente con TRIGGERS y STORED PROCEDURE

Mas información (en ingles) : http://dev.mysql.com/doc/refman/5.1/en/events.html
